# AI 编程的常识与认知

> 理解 AI 编程助手的能力边界，才能更好地驾驭它。

## 项目简介

这是一份关于 **AI 编程助手使用指南** 的文档集合，旨在帮助开发者：

- 认识 AI 编程助手的常见陷阱，避免踩坑
- 理解 AI 的工作原理和能力边界
- 掌握与 AI 协作编程的最佳实践
- 提升 AI 辅助编程的效率和代码质量

## 核心观点

| 维度 | AI 编程助手的定位 |
|------|-------------------|
| **角色** | 高效的代码生成工具，而非架构师或创新者 |
| **优势** | 快速实现已知模式，提升编码效率 |
| **局限** | 缺乏全局观、一致性意识和创新能力 |
| **最佳用法** | 明确规范 + 充足上下文 + 人工审查 |

## 常见陷阱速览

1. **上下文丢失** - 长对话中 AI 会「遗忘」早期信息
2. **缺乏项目全局观** - 重复生成已有的工具函数
3. **幻觉与虚构代码** - 编造不存在的 API
4. **过时的知识** - 推荐已被弃用的方案
5. **缺乏实现一致性** - 每次生成可能采用不同风格
6. **功能实现自由发挥** - 在缺少规范时随意选择实现方式
7. **过度工程化** - 追求「最佳实践」而忽略实际需求
8. **安全性盲区** - 可能忽略安全最佳实践

## AI 擅长的领域

- 功能函数与功能模块（边界清晰、输入输出明确）
- 模式匹配与模仿（照葫芦画瓢）
- 代码解释与文档生成
- 样板代码与配置生成
- 代码重构辅助
- Debug 辅助与错误分析
- 正则表达式与复杂语法
- 测试场景设计

## 快速参考

```
开始任务前 → 引用项目规范文档（RULES.md）
生成函数前 → 检查 utils/ 是否已有类似实现
API 调用后 → 验证方法/参数是否真实存在
依赖引入后 → 确认版本是否为最新稳定版
复杂功能时 → 拆分为：接口设计 → 实现 → 测试
代码审查时 → 重点检查一致性、重复代码、安全漏洞
```
